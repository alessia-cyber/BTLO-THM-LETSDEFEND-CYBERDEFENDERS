# TryHackMe - SOC L1 Alert Reporting


## Overview

This report details my analysis and findings from the SOC L1 Alert Reporting room on TryHackMe.
The room focuses on the critical skills required for Level 1 Security Operations Center (SOC) analysts. 
It provides a comprehensive guide and excercies on how to properly report, escalate, and communicate about high-risk security alerts.


## Tools

- Simulated SOC dashboard provided by TryHackMe.


## Skills Acquired

- Identified high-risk alerts that require immediate attention;
- Followed standard operating procedures (SOPs) for alert reporting;
- Effectively written alert comments and communicate incident details to senior analysts and stakeholders;
- Understood the proper escalation paths for different severity levels;
- Documented incidents accurately for future analysis and compliance.


## Triage Workflow and Process

1. **Initial Dashboard Review**: Scan the list of active alerts;
2. **Deep Dive into Specific Alert**: Select the alert of interest and read the details and full metadata;
3. **Escalation and Reporting**: Assign the alert to an L2 Analyst and write a report in the comment section following the Five Ws approach;
4. **Repeat for Remaining Alerts**: Repeat Steps 2 and 3 until all required alerts in the scenario have been triaged, assigned, and reported.

## Key Takeaways

- **Clarity**: Learn to write concise and accurate alert descriptions;
- **Speed**: Understand the importance of rapid escalation for high-risk threats;
- **Process**: Master the workflow from initial detection to handover.
